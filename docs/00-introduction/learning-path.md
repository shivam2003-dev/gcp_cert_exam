# 4-6 Week Learning Path

This course is designed as a structured 4-6 week self-study program. Adjust the pace based on your schedule and prior experience.

## Weekly Breakdown

### Week 1: Foundation (Modules 1-2)

**Time Commitment:** 10-15 hours

#### Module 1: Setting Up Cloud Solution Environment
- **Duration:** 4-6 hours
- **Topics:**
  - Google Cloud projects and billing
  - Enabling APIs and services
  - Installing and configuring gcloud CLI
- **Practice:** Create a project, set up billing, install gcloud
- **Key Outcomes:** Comfortable with Google Cloud Console and CLI

#### Module 2: Identity & Access Management (IAM)
- **Duration:** 6-9 hours
- **Topics:**
  - IAM fundamentals (users, groups, roles)
  - Service accounts
  - IAM policies and bindings
  - Organization policies
- **Practice:** Create service accounts, assign roles, test permissions
- **Key Outcomes:** Understand IAM hierarchy and role assignment

:::tip Study Tip
Week 1 is foundational. Don't rush through IAM - it appears throughout the exam. Spend extra time understanding service accounts and role inheritance.
:::

### Week 2: Core Infrastructure (Modules 3-4)

**Time Commitment:** 15-20 hours

#### Module 3: Networking Fundamentals
- **Duration:** 7-10 hours
- **Topics:**
  - VPCs, subnets, and regions
  - Firewall rules
  - Load balancing (HTTP(S), TCP/UDP, Internal)
  - Cloud DNS and Cloud CDN
- **Practice:** Create VPC, configure firewall rules, set up load balancer
- **Key Outcomes:** Understand Google Cloud networking architecture

#### Module 4: Compute Services
- **Duration:** 8-12 hours
- **Topics:**
  - Compute Engine (VMs)
  - Google Kubernetes Engine (GKE)
  - Cloud Run
  - App Engine
  - Cloud Functions
- **Practice:** Deploy VMs, containers, and serverless functions
- **Key Outcomes:** Know when to use each compute option

:::warning Important
Week 2 covers 40%+ of the exam content. This is the most important week. Ensure you understand the differences between compute options.
:::

### Week 3: Storage & Data (Modules 5-6)

**Time Commitment:** 12-18 hours

#### Module 5: Storage Services
- **Duration:** 6-9 hours
- **Topics:**
  - Cloud Storage (object storage)
  - Persistent Disks (block storage)
  - Cloud SQL (managed SQL)
  - Cloud Spanner (globally distributed)
  - Firestore and Bigtable (NoSQL)
- **Practice:** Create buckets, databases, and understand use cases
- **Key Outcomes:** Choose the right storage for each scenario

#### Module 6: Managed Services & Databases
- **Duration:** 6-9 hours
- **Topics:**
  - BigQuery (data warehouse)
  - Pub/Sub (messaging)
  - Cloud Endpoints (API management)
  - Additional managed services
- **Practice:** Run BigQuery queries, set up Pub/Sub topics
- **Key Outcomes:** Understand managed service offerings

:::note Remember
Storage and database questions often test your ability to choose the right service based on requirements (consistency, availability, scale, cost).
:::

### Week 4: Operations & Security (Modules 7-8)

**Time Commitment:** 12-18 hours

#### Module 7: Monitoring, Logging & Operations
- **Duration:** 6-9 hours
- **Topics:**
  - Cloud Monitoring (metrics, dashboards)
  - Cloud Logging (log management)
  - Cloud Trace (distributed tracing)
  - Error Reporting and Cloud Debugger
- **Practice:** Create dashboards, write log queries, set up alerts
- **Key Outcomes:** Monitor and troubleshoot cloud resources

#### Module 8: Security Best Practices
- **Duration:** 6-9 hours
- **Topics:**
  - Encryption (at rest and in transit)
  - Secret Manager
  - Cloud Armor (DDoS protection)
  - Binary Authorization
  - Security best practices
- **Practice:** Encrypt resources, manage secrets, review security settings
- **Key Outcomes:** Implement security best practices

:::tip Exam Tip
Operations and security questions often appear in scenario-based questions. Focus on understanding when to use each tool.
:::

### Week 5: Deployment & Review (Module 9)

**Time Commitment:** 10-15 hours

#### Module 9: Deployment & CI/CD
- **Duration:** 6-9 hours
- **Topics:**
  - Cloud Build (CI/CD)
  - Deployment Manager
  - Terraform basics
  - Cloud Source Repositories
- **Practice:** Set up CI/CD pipeline, deploy infrastructure as code
- **Key Outcomes:** Deploy applications and infrastructure

#### Review Week
- **Duration:** 4-6 hours
- **Activities:**
  - Review all modules
  - Revisit practice questions you got wrong
  - Create summary notes
  - Identify weak areas

:::warning Common Pitfall
Don't skip the review week. Consolidating knowledge is crucial for exam success.
:::

### Week 6: Practice Exams & Final Prep

**Time Commitment:** 10-15 hours

#### Practice Exams
- **Duration:** 8-12 hours
- **Activities:**
  - Take full-length practice exam 1
  - Review all answers and explanations
  - Take full-length practice exam 2
  - Review all answers and explanations
  - Focus study on weak areas

#### Final Review
- **Duration:** 2-3 hours
- **Activities:**
  - Quick review of key concepts
  - Review IAM roles and policies
  - Review CLI commands
  - Review service selection criteria

:::tip Exam Tip
Aim for 80%+ on practice exams before scheduling the real exam. If you're consistently scoring lower, spend more time reviewing weak areas.
:::

## Daily Study Schedule

### Recommended Daily Routine

**Option 1: Intensive (4 weeks)**
- **Weekdays:** 2-3 hours/day
- **Weekends:** 4-6 hours/day
- **Total:** ~15-20 hours/week

**Option 2: Moderate (6 weeks)**
- **Weekdays:** 1-2 hours/day
- **Weekends:** 3-4 hours/day
- **Total:** ~10-12 hours/week

### Study Session Structure

1. **Review previous day** (10 minutes)
2. **Read new content** (60-90 minutes)
3. **Hands-on practice** (30-60 minutes)
4. **Practice questions** (30-45 minutes)
5. **Review answers** (15-30 minutes)

## Adjusting the Schedule

### If You Have More Time (8+ weeks)
- Spend more time on hands-on practice
- Explore additional Google Cloud services
- Build a sample project
- Deep dive into areas of interest

### If You Have Less Time (3-4 weeks)
- Focus on high-weight domains (3, 4, 2)
- Prioritize practice questions
- Use practice exams to identify gaps
- Focus study on weak areas

### If You Have Prior Cloud Experience
- You may move faster through early modules
- Focus on Google Cloud-specific differences
- Emphasize practice questions
- Take practice exams earlier
