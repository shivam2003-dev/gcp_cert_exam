# Course Overview

## Course Objectives

By the end of this course, you will be able to:

- Set up and manage Google Cloud projects and billing accounts
- Configure identity and access management (IAM) policies
- Deploy and manage compute resources (VMs, containers, serverless)
- Configure networking, including VPCs, firewall rules, and load balancers
- Implement storage solutions (object storage, block storage, databases)
- Monitor, log, and troubleshoot cloud resources
- Implement security best practices
- Deploy applications using CI/CD pipelines

## Exam Domains Breakdown

The ACE exam is organized into the following domains:

### Domain 1: Setting up a cloud solution environment (15%)

- Setting up cloud projects and accounts
- Managing billing configuration
- Installing and configuring the command line interface (CLI), specifically the Cloud SDK

### Domain 2: Planning and configuring a cloud solution (20%)

- Planning and estimating Google Cloud product use
- Planning and configuring compute resources
- Planning and configuring data storage options
- Planning and configuring network resources

### Domain 3: Deploying and implementing a cloud solution (25%)

- Deploying and implementing Compute Engine resources
- Deploying and implementing Kubernetes Engine resources
- Deploying and implementing Cloud Run and Cloud Functions
- Deploying and implementing data solutions
- Deploying and implementing networking resources

### Domain 4: Ensuring successful operation of a cloud solution (25%)

- Managing Compute Engine resources
- Managing Kubernetes Engine resources
- Managing Cloud Run and Cloud Functions
- Managing networking resources
- Monitoring and logging

### Domain 5: Configuring access and security (15%)

- Managing identity and access management (IAM)
- Managing service accounts
- Viewing audit logs

:::tip Exam Tip
Notice that Domains 3 and 4 together make up 50% of the exam. Focus heavily on deploying and managing resources, as this is the core of the ACE role.
:::

## Learning Approach

This course uses a **practical, exam-focused approach**:

1. **Conceptual Understanding:** Learn what each service does and when to use it
2. **Hands-On Practice:** Use Google Cloud Console and gcloud CLI
3. **Exam Patterns:** Recognize common question patterns and scenarios
4. **Practice Questions:** Test understanding with realistic exam-style questions

:::warning Common Pitfall
Don't just read the content passively. The ACE exam tests your ability to perform tasks, not just recall information. Practice every command and configuration shown in the examples.
:::

## Study Tips

### 1. Active Learning
- Type out every CLI command yourself
- Create resources in Google Cloud Console
- Experiment with different configurations
- Break things and fix them

### 2. Note-Taking
- Create a study guide as you progress
- Note common patterns and "gotchas"
- Document CLI commands you use frequently
- Keep a list of service limits and quotas

### 3. Practice Questions
- Complete all practice questions in each module
- Review answer explanations carefully
- Understand why wrong answers are wrong
- Retake practice questions you got wrong

### 4. Time Management
- Follow the 4-6 week schedule
- Spend 1-2 hours daily on study
- Reserve the last week for practice exams
- Take breaks to avoid burnout

:::note Remember
The ACE exam is associate-level, meaning it tests practical skills, not deep architectural knowledge. Focus on "how to do" rather than "why it works this way."
:::

## Google Cloud Free Tier

To practice hands-on, you'll need a Google Cloud account:

- **Free Tier:** Always free services (with usage limits)
- **Free Trial:** $300 credit for 90 days
- **Always Free:** Limited but permanent free tier

:::tip Exam Tip
Set up billing alerts immediately to avoid unexpected charges. The exam will test your knowledge of billing and cost management.
:::
